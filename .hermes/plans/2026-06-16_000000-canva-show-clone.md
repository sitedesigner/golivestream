### Task 6: Build the Canva API integration script

**Objective:** Create `scripts/canva-show-cloner.py` — clones a Canva template design for new shows.

**File:** `scripts/canva-show-cloner.py`

**What it does:**
1. Authenticates with Canva OAuth 2.0 (token stored in env var `CANVA_TOKEN`)
2. Reads a base template design ID from config
3. For each show (BTC, Beyond the Cart, TDDS):
   - Clones the template design
   - Replaces text placeholders (show name, host name, colors)
   - Updates brand colors per show
   - Exports as PNG (thumbnail) and PDF (one-pager)
   - Saves to `assets/{show-name}/`

**Config structure (JSON):**
```json
{
  "template_design_id": "DAFxxxxxx",
  "shows": {
    "winsday": {
      "name": "WinsDay",
      "host": "Alanna Smith",
      "primary_color": "#0D1B3E",
      "accent_color": "#F5A623",
      "streamyard_account": "winsday"
    },
    "btc": {
      "name": "BTC",
      "host": "David Goecke",
      "primary_color": "#1A1A2E",
      "accent_color": "#F7931A",
      "streamyard_account": "btc"
    },
    "beyond-the-cart": {
      "name": "Beyond the Cart",
      "host": "David Goecke",
      "primary_color": "#0D1B3E",
      "accent_color": "#1EA56B",
      "streamyard_account": "btc"
    },
    "tdds": {
      "name": "The David Daily Show",
      "host": "David Goecke",
      "primary_color": "#0D1B3E",
      "accent_color": "#F5A623",
      "streamyard_account": "tdds"
    }
  }
}
```

**Prerequisites the user needs to do:**
1. Create a Canva app at https://www.canva.dev/apps/
2. Get OAuth 2.0 credentials (client ID + client secret)
3. Generate a refresh token
4. Store in env vars: `CANVA_CLIENT_ID`, `CANVA_CLIENT_SECRET`, `CANVA_REFRESH_TOKEN`
5. Create a template design in Canva with placeholder text elements
6. Share the design ID

**Verification:**
```bash

python3 scripts/canva-show-cloner.py --dry-run
# Should print show list and planned actions without errors
```

## Risks, Tradeoffs, and Open Questions

1. **Canva API rate limits** — Canva's API has rate limits. Cloning 4 designs in a row should be fine, but bulk operations need pacing.
2. **Canva template design** — User needs to create the base template in Canva first. The script assumes text elements with specific placeholder names.
3. **GHL Google Calendar sync** — Still needs Google OAuth setup (separate from this plan). The native GHL integration is the no-code path.
4. **GitHub auth** — `gh` CLI device flow keeps timing out. User needs to provide a PAT or create the repo manually.
5. **Show config accuracy** — The `config/shows.json` values (colors, times, hosts) are estimates. User needs to verify each show's actual settings.
